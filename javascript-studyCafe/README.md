# 스터디 카페

## 기능 요구 사항

### 1. 초기 데이터 설정

프로그램 시작 시 스터디를 정보와 기존 예약 현황을 로드해야 한다.

#### 스터디룸 목록

- 라벤더 (2인실): 1시간당 5,000원
- 세이지 (4인실): 1시간당 8,000원
- 페퍼민트 (6인실): 1시간당 12,000원

#### 운영 시간

- 평일 (월~금): 10:00 ~ 22:00
- 주말 (토~일): 10:00 ~ 24:00 (밤 12시)

#### 기존 예약 현황

- 2024-06-15 (토)|페퍼민트|14:00~17:00
- 2024-06-17 (월)|라벤더|18:00~20:00

### 2. 예약 진행 프로세스

1. 예약 날짜 입력: 방문한 날짜를 `YYYY-MM-DD` 형식으로 입력받는다.
   - 과거 날짜거나 형식이 올바르지 않으면 에러 처리한다. (오늘 날짜 6/1로 가정)
2. 스터디룸 선택: 예약할 룸 이름을 입력받는다.
3. 이용 시간 입력: 시작 시간과 종료 시간을 `HH:MM-HH:MM` 형식으로 입력받는다.
   - 유효성 검사 1: 운영 시간 내여야 한다.
   - 유효성 검사 2: 시작 시간은 종료 시간보다 앞서야 하며, 최소 1시간 이상 이용해야 한다.
   - 유효성 검사 3 (중복 확인): 기존 예약 시간과 1분이라도 겹치면 예약할 수 없다.
4. 인원 수 입력: 입장할 인원 수를 입력받는다.
   - 룸의 수용 인원을 초과하면 에러 처리한다.

### 3. 요금 계산

1. 기본 요금: `룸 시간당 단가` x `이용 시간`
2. 할증 정책 (중복 허용)
   - 주말 할증: 토, 일요일 예약 시 총액의 10% 가산
   - 야간 할증: 20:00 이후의 이용 시간에 대해서는 해당 시간분 만큼 시간당 단가의 10% 가산
     - 예: 19:00 ~ 21:00 (2시간) 이용 시, 19~20시는 기본, 20~21시는 야간 할증 적용
3. 할인 정책
   - 장시간 할인: 4시간 이상 예약 시 총 요금(할증 포함)의 5% 할인
4. 최종 결제: 견적서 출력

### 4. 입출력 예시

#### 실행 결과

```bash
안녕하세요! 우아한 스터디 카페입니다.
예약하실 날짜를 입력해 주세요. (YYYY-MM-DD)
2024-06-15

예약하실 스터디룸을 입력해 주세요. (라벤더, 세이지, 페퍼민트)
페퍼민트

이용 시간을 입력해 주세요. (HH:MM-HH:MM)
20:00-24:00

사용 인원을 입력해 주세요.
5

==============================
<예약 확인>
날짜: 2024-06-15 (토)
방: 페퍼민트 (6인실)
시간: 20:00 ~ 24:00 (4시간)
인원: 5명

<결제 내역>
기본 요금: 48,000원
주말 할증(+10%): +4,800원
야간 할증(4시간): +4,800원
장시간 할인(-5%): -2,880원

최종 결제 금액: 54,720원
==============================
```

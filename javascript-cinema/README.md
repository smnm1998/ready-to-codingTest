# javascript-cinema

## 기능 요구 사항

### 1. 초기 데이터 설정

프로그램 시작 시 상영 영화 정보와 상영 스케줄을 미리 로드해야 한다.

#### 영화 목록

- 아바타 2: 180분 | 15,000원
- 서울의 봄: 140분 | 14,000원
- 노량: 150분 | 14,000원

#### 상영 스케줄 및 좌석 현황

- 각 영화는 하루에 한 번만 상영한다고 가정
- 좌석은 1열(A)부터 3열(C)까지, 각 열당 5좌석(1~5)이 있다.
- 아바타 2: 09:00 상영 | 예매된 좌석: A1, A2
- 서울의 봄: 13:00 상영 | 예매된 좌석: 없음
- 노량: 19:00 상영 | 예매된 좌석: C4, C5

### 2. 영화 예매 진행

1. 영화 목록 출력: 상영 중인 영화의 제목, 상영 기간, 가격을 출력한다.
2. 영화 선택: 예매할 영화의 제목을 입력받는다.
3. 좌석 확인: 선택한 영화의 상영 시간과 예매 가능한 좌석 배치도를 출력한다.
   - 예매된 좌석은 X, 예매 가능한 좌석은 O로 표시한다.
4. 좌석 선택: 예매할 좌석 번호를 입력받는다. (예: `A3`)
   - 이미 예매된 좌석이거나 존재하지 않는 좌석을 선택하면 에러 메시지를 출력하고 다시 입력받는다.
5. 할인 적용
   - 조조 할인: 10:00 이전 상영 영화는 2,000원 할인된다.
   - 멤버십 할인: 멤버십 할인을 받을지 여부(Y/N)를 입력받는다.
   - 할인은 중복 적용이 가능하다.
6. 최종 결제: 할인 내역과 최종 결제 금액을 출력한다.

### 3. 예외 처리

- 사용자가 잘못된 값을 입력한 경우, `[ERROR]`로 시작하는 메시지와 함께 `Error`를 발생시키고 해당 메시지를 출력한 다음 해당 지정부터 다시 입력을 받는다.

## 기능 구현 목록

### Views

1. InputView.js - 입력 관리
2. OutputView.js - 출력 관리

### Models

1. Discount.js - 할인 관리
2. Cinema.js - 영화 및 좌석 관리
3. InputParse.js - 입력값 파싱
4. Calculator.js - 결제 관리
5. Validator.js - 검증 및 에러 관리

### Constants

1. Movie.js - 영화 정보 객체
2. Seat.js - 좌석 객체

### etc.

- App.js를 컨트롤러로 사용
- 에러 발생 시, 에러난 시점의 입력 부분을 다시 받도록 만들 것

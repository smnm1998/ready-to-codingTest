# javascript-cafe

## 기능 요구 사항

우아한 카페에 오신 것을 환영합니다. 손님들이 주문한 메뉴와 수량을 입력받아, 할인 정책을 적용한 후 최종 결제 금액과 영수증을 출력하는 키오스크 시스템을 구현해야 합니다.

### 1. 메뉴 및 가격 정보

[음료]

- 아메리카노: 4,500원
- 카페라떼: 5,000원
- 오렌지주스: 5,500원

[디저트]

- 초코케이크: 6,500원
- 치즈케이크: 7,000원

### 2. 주문 프로세스

- 식사 장소 선택: 매장 또는 포장 중 하나를 입력받습니다.
- 메뉴 주문: 메뉴명-수량 형식을 쉼표(,)로 구분하여 입력받습니다. (예: 아메리카노-2,초코케이크-1)
- 주문 확인 및 할인 적용: 입력된 주문을 바탕으로 할인 혜택을 계산합니다.
- 영수증 출력: 주문 내역, 할인 내역, 최종 결제 금액을 출력합니다.

### 3. 할인 정책

- 포장 할인: 포장 선택 시 음료 메뉴 1잔당 500원 할인 (디저트 제외).
- 디저트 할인: 음료와 디저트를 함께 주문할 경우, 디저트 1개당 1,000원 할인.
  - (예: 음료 2잔, 디저트 1개 -> 디저트 1,000원 할인)
  - (예: 음료 1잔, 디저트 2개 -> 디저트 1,000원 할인 (1:1 매칭))
- 통신사 할인: 할인 전 총주문 금액의 5% 할인. (최대 2,000원 한도)
  - 통신사 할인 적용 여부를 Y/N으로 입력받습니다.

### 4. 입력 예외 처리 ([ERROR]로 시작하는 에러 메시지 출력 후 재입력)

- 존재하지 않는 메뉴를 입력한 경우
- 메뉴 형식이 예시와 다른 경우 (예: 아메리카노2개)
- 메뉴 개수가 1개 미만인 경우
- 중복 메뉴를 입력한 경우 (예: 아메리카노-1,아메리카노-1)
- 총주문 수량이 20개를 초과하는 경우

## 입출력 요구 사항

### 입력 예시 1

```bash
식사 장소를 선택해 주세요. (매장/포장)
포장

주문하실 메뉴와 수량을 입력해 주세요. (예: 아메리카노-2,초코케이크-1)
아메리카노-2,치즈케이크-1

통신사 제휴 할인을 적용하시겠습니까? (Y/N)
Y
```

### 입력 예시 2

```bash
[주문 내역]
아메리카노 2개
치즈케이크 1개

[할인 전 총금액]
16,000원

[할인 내역]
포장 할인: -1,000원
디저트 할인: -1,000원
통신사 할인: -800원

[최종 결제 금액]
13,200원
```

## 프로그래밍 요구 사항

- Node.js 18.17.1 이상에서 실행 가능해야 합니다.
- ApplicationTest.js 파일의 테스트가 모두 통과해야 합니다.
- Indent(들여쓰기) depth를 2까지만 허용합니다.
- 3항 연산자를 쓰지 않습니다.
- 함수(또는 메서드)의 길이가 15라인을 넘어가지 않도록 구현합니다.
- else 예약어를 쓰지 않습니다.
- @woowacourse/mission-utils의 Console API를 사용하여 입출력을 구현해야 합니다.
- 사용자가 잘못된 값을 입력할 경우 throw new Error("[ERROR] 메시지")를 발생시키고, 에러 메시지를 출력 후 해당 지점부터 다시 입력을 받아야 합니다.

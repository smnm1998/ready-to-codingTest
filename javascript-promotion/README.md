# 🎄 12월 이벤트 플래너

우아한테크코스 최종 코딩 테스트 모의고사 미션입니다.

## 📝 미션 목표

레스토랑에서 12월 동안 진행되는 이벤트를 기획하고, 고객의 방문 날짜와 주문 메뉴를 입력받아 적용 가능한 모든 혜택을 계산하여 총혜택 금액과 최종 결제 금액을 출력하는 프로그램을 구현합니다.

객체지향 생활 체조 원칙을 준수하며, 도메인 로직을 명확하게 분리하고, 예외 처리를 완벽하게 구현하는 것이 중요합니다.

## 📋 기능 요구사항

### 1. 입출력 및 실행

- `App` 클래스의 `run` 메서드를 통해 애플리케이션을 실행해야 합니다.
- 사용자로부터 **방문 예정일(1~31)**과 **주문 메뉴**를 입력받습니다.
- 모든 출력은 터미널 입/출력 예시의 순서와 형식을 정확히 따릅니다.

### 2. 메뉴판

| 구분           | 메뉴 및 가격                                                                                  |
| :------------- | :-------------------------------------------------------------------------------------------- |
| **애피타이저** | 양송이수프 (6,000) / 타파스 (5,500) / 시저샐러드 (8,000)                                      |
| **메인**       | 티본스테이크 (55,000) / 바비큐립 (54,000) / 해산물파스타 (35,000) / 크리스마스파스타 (25,000) |
| **디저트**     | 초코케이크 (15,000) / 아이스크림 (5,000)                                                      |
| **음료**       | 제로콜라 (3,000) / 레드와인 (60,000) / 샴페인 (25,000)                                        |

### 3. 이벤트 및 혜택 로직

#### 3.1. 할인 및 증정 조건

- **할인 공통 조건:** 할인 전 총주문 금액이 **12,000원 이상**일 때만 적용됩니다.
- **크리스마스 디데이 할인:** 12월 1일(1,000원)부터 25일까지 매일 100원씩 증가.
- **평일 할인:** 월~금요일, 디저트 메뉴 1개당 **2,025원** 할인.
- **주말 할인:** 토~일요일, 메인 메뉴 1개당 **2,025원** 할인.
- **특별 할인:** 별(★)이 있는 날 (3, 10, 17, 24, 25, 31일), 총주문 금액에서 **1,000원** 할인.
- **증정 이벤트:** 할인 전 총주문 금액이 **120,000원 이상**일 때, **샴페인 1개 (25,000원)** 증정.

#### 3.2. 이벤트 배지

총혜택 금액(증정 메뉴 가격 포함)에 따라 배지 부여.

- 20,000원 이상: **산타**
- 10,000원 이상: **트리**
- 5,000원 이상: **별**
- 그 외: **없음**

### 4. 예외 처리

사용자가 잘못된 값을 입력할 경우, `[ERROR]` 문구와 함께 해당 부분부터 입력을 다시 받습니다.

- **방문 예정일:** 1~31 사이의 숫자가 아닌 경우.
- **주문 메뉴:**
  - 메뉴판에 없는 메뉴 주문 시
  - 메뉴-개수 형식이 아닌 경우
  - 총 주문 수량이 **20개를 초과**하는 경우
  - 메뉴를 **중복**해서 주문하는 경우
  - **음료만** 주문하는 경우
  - 개수가 1 이상이 아닌 경우

## 🛠️ 프로그래밍 요구사항 및 코드 컨벤션

### 1. 객체지향 설계 및 모듈 분리

- **도메인 로직 분리:** 날짜, 주문, 할인 계산 등 도메인 로직은 별도의 클래스/객체로 분리하여 `App` 클래스의 로직을 최소화합니다.
- **View 분리:** 입력은 `InputView`, 출력은 `OutputView`와 같이 View 역할을 하는 모듈을 분리하여 구현합니다.
- **상수 관리:** 메뉴 정보, 에러 메시지, 이벤트 기준 등은 상수를 활용하여 관리합니다.
- **함수 분리:** 함수(메서드)는 하나의 기능만 수행하도록 작게 만들고, 이름은 의도를 명확히 드러내야 합니다.

### 2. 코드 컨벤션 (필수 준수)

- **ES6+ 사용:** `const`, `let` 키워드를 사용하고, `var`는 사용하지 않습니다.
- **들여쓰기:** **스페이스 2칸**을 사용합니다.
- **Naming Convention:**
  - 클래스: **파스칼 케이스** (`EventPlanner`)
  - 변수/함수: **카멜 케이스** (`calculateDiscount`)
  - 상수: **스네이크 케이스** (대문자) (`MIN_ORDER_FOR_EVENT`)
- **객체 접근:** 객체의 필드에 직접 접근을 지양하고 **Getter 메서드**를 사용합니다.
